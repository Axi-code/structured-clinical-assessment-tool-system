# 截图目录

本目录保留为项目截图占位目录，便于兼容旧文档中的截图路径写法。

如果你后续统一使用 `docs/images/`，也可以把新截图优先放到 `docs/images/` 中；
如果已有文档引用了 `docs/screenshots/`，则可继续在本目录存放页面截图。
# 项目截图说明

本目录用于存放项目演示截图，便于：

- 在 `README.md` 中展示系统界面
- 在答辩 PPT 或说明书中引用
- 在 GitHub 仓库中直观展示项目成果

## 推荐截图内容

- 登录页
- 首页统计看板
- 患者管理页面
- 科室管理页面
- 诊断管理页面
- **诊断详情页**（含 AI 建议诊断、一键加入字典、确认诊断）
- 评估模板管理页面
- 评估规则管理页面
- 评估填写页面
- 历史评估记录页面
- **AI 诊疗建议弹窗**
- 报告预览页面
- 数据统计页面
- 用户管理页面
- 操作日志页面

## 推荐命名方式

建议采用统一命名，便于管理，例如：

- `login.png`
- `dashboard.png`
- `patient-list.png`
- `template-list.png`
- `assessment-form.png`
- `report-preview.png`

## 使用建议

- 尽量使用统一分辨率截图
- 避免在截图中暴露真实隐私数据
- 可以优先展示最能体现项目特色的页面，例如统计、评估、AI 建议、报告导出等
