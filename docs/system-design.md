# 系统设计说明书

## 1. 项目概述

《结构化临床评估工具系统》是一个面向医院临床场景的前后端分离式系统，用于支持患者信息管理、结构化评估、评估规则计算、报告生成、统计分析以及 AI 辅助诊疗建议生成。

系统目标包括：

- 提升临床评估流程的标准化与结构化程度
- 减少人工录入与人工计算带来的错误
- 形成可回溯的评估记录、报告与操作日志
- 为医生、护士、管理员提供分角色的业务支撑
- 为后续扩展智能评估、风险预警和数据分析打下基础

## 2. 系统角色

系统当前包含三类核心角色：

- `ADMIN`：管理员，负责用户、科室、诊断、模板、日志等全局管理
- `DOCTOR`：医生，负责患者评估、规则配置、报告查看、诊疗建议生成等业务
- `NURSE`：护士，参与患者管理、评估操作、统计查看及部分 AI 相关流程

## 3. 总体架构

系统采用前后端分离架构。

- 前端：基于 Vue 3 + Vite + Element Plus 构建管理端页面
- 后端：基于 Spring Boot + MyBatis-Plus 提供 REST API
- 数据库：使用 MySQL 8 保存业务数据
- AI 能力：通过 Qwen 接口实现模板推荐、对话式评估、诊疗建议生成

## 5. 功能模块设计

### 5.1 用户与权限模块

主要功能：

- 登录、退出、刷新会话
- 用户新增、更新、删除、查询
- 密码修改
- 角色权限控制
- 验证码与登录限流

设计说明：

- 使用 Sa-Token 进行登录态管理
- Access Token 放在请求头中
- Refresh Token 放在 HttpOnly Cookie 中
- 管理员拥有更多系统管理能力

### 5.2 患者管理模块

主要功能：

- 患者列表分页查询
- 患者新增、编辑、删除
- 患者与科室、诊断关系维护
- 历史评估记录查看

设计说明：

- 患者基础信息中包含敏感字段
- 身份证号、电话、地址、紧急联系人等字段支持加密存储
- 患者与科室、诊断采用关联 ID 方式维护

### 5.3 评估模板模块

主要功能：

- 评估模板新增、编辑、删除
- 评估字段维护
- 模板启停用
- 模板版本管理
- 模板与科室适用关系配置

设计说明：

- 模板支持版本号
- 字段、规则与模板之间是一对多关系
- 通过 `template_department` 表维护模板适用科室

### 5.4 评估规则与记录模块

主要功能：

- 规则列表与详情查询
- 规则新增、更新、删除
- 规则测试
- 实时计算
- 评估草稿创建、保存、提交
- 历史记录查询与多记录对比

设计说明：

- 规则支持评分、风险、计算等类型
- 提交评估后可自动计算总分、风险等级与提示信息
- 当规则不可用时，可降级使用 AI 结果兜底

### 5.5 报告与统计模块

主要功能：

- PDF / Word 预览与导出
- 报告模板管理
- 默认报告模板设置
- 看板统计、按科室统计、按模板统计、风险预警等

设计说明：

- 报告模板独立维护，和评估模板进行关联
- 统计模块面向管理与业务监控场景

### 5.6 AI 能力模块

主要功能：

- 对话式评估
- AI 推荐评估模板
- AI 生成模板草稿
- AI 实时辅助计算
- AI 生成诊疗建议与重生成

设计说明：

- AI 模块依赖外部大模型接口
- 当未配置 API Key 时，相关能力不可用
- 适合用于提升评估效率与辅助临床决策

## 6. 数据流设计

以典型评估流程为例：

1. 用户登录系统并获取访问令牌。
2. 选择患者，查看患者基础信息和所属科室。
3. 根据科室或 AI 推荐选择合适的评估模板。
4. 创建评估草稿并录入结构化评估数据。
5. 调用规则引擎或 AI 计算评估结果。
6. 保存或提交评估记录。
7. 根据评估结果生成报告或诊疗建议。
8. 系统记录操作日志，用于审计和追溯。

## 7. 安全设计

系统当前具备以下安全设计：

- 基于 Sa-Token 的登录态与接口鉴权
- 基于角色的访问控制
- 登录验证码与限流保护
- Refresh Token 使用 HttpOnly Cookie
- 患者敏感字段支持 AES 加密
- 操作日志记录关键行为
- 逻辑删除避免误删后直接丢失数据

## 8. 可扩展性设计

系统具备如下扩展方向：

- 将 Refresh Token 从内存存储迁移到 Redis
- 增加 Docker / Docker Compose 部署方式
- 引入 Flyway 或 Liquibase 统一数据库迁移
- 补充自动化测试与更细粒度接口文档
- 扩展更多临床评估模板与 AI 能力

## 9. 总结

本系统围绕临床评估场景进行设计，兼顾了业务管理、数据留痕、权限控制、报告导出和 AI 辅助能力。整体结构清晰，适合作为毕业设计项目展示，也具备继续工程化演进的基础。
